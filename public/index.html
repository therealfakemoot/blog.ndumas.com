Prometheus Primer

2019-07-04
#prometheus 
Querying Basics Queries run against metrics, which are sets of timeseries data. They have millisecond granularity and are stored as floating point values. Using Queries Queries reference individual metrics and perform some analysis on them. Most often you use the rate function to “bucket” a metric into time intervals. Once the metric in question has been bucketed into time intervals, you can do comparisons. (rate(http_response_size_bytes[1m])) > 512 This query takes the size of http responses in bytes and buckets it into one minute intervals and drops any data points smaller than 512 bytes.
